1. Частично создана отология предметной области (ПрО) АНПА.
1.1 АНПА является базовым классом в иерархии наследования сущностей ПрО, все остальные происходят из нее.
Этот подход (если верить руководствам) позволяет отделить сущности в свое пространство имен и в последствии объединять несколько онтологий в одну.
1.2 Показаны взаимосвязи типа (является частью/состоит из) для материальных объектов декомпозированой модели АНПА.

2. Положено начало описания типов данных для протокола по шине данных Modbus.
2.1 Введено понятие ModbusElement, для использования в триплетах вида "АНПА - СоотнесенС - ModbusElement".
2.1.1 Определен тип данных modbus:TransmitedType = {"Bool" , "DoubleWord" , "LongReal" , "Real" , "Word"}.
2.1.2 Определены DataProperty для ModbusElement (title, address, type), где type это один из modbus:TransmitedType.

3. Таким образом становится возможным завести индивидуальности (экземпляры классов) такие как:
3.1 пу_U^^ModbusElement - которому в полях `Data property assertions` поставлены в соответствие значения:
    address = "42"^^xsd:short
    type = "Real"^^modbus::TransmitedType
    title = "A1"^^xsd:string
3.2 пу_U_value^^ModbusData - наследник ModbusElement с полем значение ("ModbusElement - Значение - ModbusData")
    value = "27"^^xsd::anyURI
Тип поля "значение" пока непонятно каким ставить. У меня в программе это полиморфный тип данных, который может быть конвертирован во все базовые типы данных.
